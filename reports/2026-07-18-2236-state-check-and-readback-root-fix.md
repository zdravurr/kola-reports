## 2. 🔴 КОРЕНЬ ЛОЖНЫХ ТРЕВОГ — НАЙДЕН И ПОЧИНЕН (главное за сессию)

Сторож `post_send_check` шесть раз подряд заорал «БРАК УШЁЛ ПОКУПАТЕЛЮ / landed text DIFFERS»
на НОРМАЛЬНО доставленных сообщениях (JoeJoe, Samantha, Analilia, Ramiro, Maria, john).

**Диагноз зависшей сессии был НЕВЕРЕН.** Она решила «не долистали до низа треда» и написала
докрутку. Докрутка вдобавок стояла под `if not dump_fn`, а боевой вызов
(`offerup_send._compose_and_send`) передаёт `dump_fn=dump` — то есть на реальном пути
отправки не выполнялась НИ РАЗУ (живой промах: тред «Price Negotiable» 22:21).

**Настоящий корень — ПОРЯДОК РАЗБОРА.** Замер координат в треде сразу после отправки:

```
docidx=7   y=1481  ME: «...сектионал уже ушёл»                (НОВОЕ, 06:21 PM)
docidx=10  y=963   ME: «Hi Daniel! Yes, it is still available» (СТАРОЕ, 01:48 AM Jul 10)
```

На ЭКРАНЕ порядок правильный (старое вверху, новое внизу). А в XML-дереве свежий пузырь
идёт РАНЬШЕ старого. `_ou_labeled` возвращала порядок обхода дерева, поэтому `mine[-1]`
(«последнее наше») брала САМЫЙ СТАРЫЙ ответ. Треды при этом целиком помещались на экране —
листать было нечего.

**Почему это не «лишний шум»:** этот же перекос читает анти-дубль («последнее слово наше»),
то есть это money-path. И сторож, который орёт впустую, приучает себе не верить — в тот раз,
когда текст ДЕЙСТВИТЕЛЬНО исказится, ему не поверят.

**Починка (коммит `c595085`):** `_ou_labeled` сортирует пузыри по Y (контракт функции —
«старые→новые»). Докрутка оставлена как отдельная гарантия для ДЛИННЫХ тредов, выведена
из-под `if not dump_fn`, с честным комментарием, что это НЕ тот корень.

**ДОКАЗАНО ЖИВЬЁМ** на треде «Price Negotiable» после починки:
```
LAST OUTGOING → «...the sofa's already gone...»            (наше НОВОЕ, верно)
PREV OUTGOING → «Hi Daniel! Yes, it is still available...» (старое, верно)
```
Коммит прошёл через commit-gate ПО ТОКЕНУ (money-path), тревога Боссу ушла (message_id 1115).
