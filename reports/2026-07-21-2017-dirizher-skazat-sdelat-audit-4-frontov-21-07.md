## ЧТО СДЕЛАНО (коммит 57cee5a → origin/master, гейт-токен, Босс уведомлён диффом)

**`lib/promise_tracker.py` (новый).**
- `extract_promises()` — ловит намерения 1-го лица (вернусь/доложу/проверю/аудит/разберусь/
  уточню/выставлю/ответ покупателю/починю/пост-прогрев/настрою/сделаю), **отсекая отрицания**
  («не буду/не могу» → НЕ обещание) и чистые ack («Понял, разбираю» → пусто).
- `capture()` — зовётся из пути ответа Дирижёра **ДО отправки**; каждое обещание → обязательство
  в `commitments.json` с РЕАЛЬНЫМ дедлайном (класс → 30/45/60/120/180 мин; явный срок в тексте
  перебивает) и исполнителем.
- `tick()` (крон `*/15`) — просроченное открытое обещание → **само-эскалация Боссу** через
  `fault_realert` по лестнице **1ч/3ч/12ч**, срочность по домену (money 🚨 > social > infra).
- `fulfill()` — гаснет ТОЛЬКО по факту выполнения (снимает и обязательство, и эскалацию).

**Проводка.** `dirigent_voice.handle_command` — `capture()` в ЕДИНОЙ точке отправки: каждый ответ
Боссу/Зине сканируется. `dirigent_brain._PROMPT` — **гейт честности**: обещай только конкретное
действие с дедлайном; не можешь — скажи прямо, НЕ обещай. promise_tracker — backstop: любое
проскочившее обещание всё равно поймано и проверено по сроку.

**E2E доказан in-process (без телефона, без спама Боссу):** сказал «проверю…вернусь…выставлю» →
завёл **3** задачи → искусственная просрочка → эскалация ушла бы Боссу как «🚨 ОБЕЩАНИЕ НЕ
ВЫПОЛНЕНО (деньги)» → `fulfill()` погасил → повторный tick тих. Тестовые строки убраны из леджера.
Поймал и починил свой баг: кириллический ярлык давал пустой ascii-слаг → все обещания в один id
(коллизия) → хеш ярлыка.
