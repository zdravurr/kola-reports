## 1. ОДНО ОКНО — ДОКАЗАНО

Новый `lib/ogon_gate.py` = ОДИН ответ на «можно ли Огню сейчас» = **окно И полоса**.
Он НЕ второй источник окна: границы и зона по-прежнему только в `warmup_window`,
ogon_gate их применяет вместе с пультом. Иначе четыре вызывающих сложили бы два условия
каждый по-своему = четыре новых копии правила, и они разъехались бы так же.

**Живая проверка — все ланы держат ОДИН объект модуля:**

    источник окна: lib/warmup_window 06:00-22:00 America/New_York
    X fin/ca   -> ww.within      (тот же модуль)
    FB прогрев -> ogon_gate -> ww
    FB/EV      -> ogon_gate -> ww
    живой вердикт всех: True
    fb_active.warmup_window id 0x7e9a080e0c20 == warmup_window id  → СОВПАДАЕТ

**Граница окна — старое FB-правило против нового:**

     ET время | окно | ogon_gate | СТАРОЕ FB-правило (08-20 PT, PDT=-7)
     05:59 ET | False | False    | False (было 02h PT)
     06:00 ET | True  | True     | False (было 03h PT)  <-- РАСХОЖДЕНИЕ
     07:30 ET | True  | True     | False (было 04h PT)  <-- РАСХОЖДЕНИЕ
     12:00 ET | True  | True     | True
     21:59 ET | True  | True     | True  (было 18h PT)
     22:00 ET | False | False    | True  (было 19h PT)  <-- РАСХОЖДЕНИЕ
     23:00 ET | False | False    | True  (было 20h PT)  <-- РАСХОЖДЕНИЕ

То есть FB реально постил до 23:00 ET (час после отбоя) и спал с 06 до 11 ET.

**FB/EV РАЗРЕШЁН ВНУТРИ ОКНА — не словами, а логом живого прогона.** Запустил настоящий
`fb_active.py session`, он прошёл гейт, взял tab_lock, поднял katana и пролистал ленту:

    {"ts":"2026-07-17T11:06:55Z","kind":"browse","scrolls":9,"likes":0}

11:06:55Z = **07:06 ET = 04:06 PT**. Старое правило (`8<=h<=20 PT`) в этот момент написало бы
`skip_night`. Оно и писало — каждое утро. Это и есть доказательство, что пояс починен.

**Ноябрьский баг убит попутно:** `timezone(timedelta(hours=-7))` — это PDT намертво; при
PST «08-20 PT» стало бы фактическими 07-19 PT. Плюс `.sh` считал по НАСТОЯЩЕЙ зоне
(`TZ=America/Los_Angeles`), а запускаемый им `.py` — по смещению: зимой они разъехались бы
ещё и между собой. Обеих копий больше нет:

    grep "hours=-7|TZ=America/Los_Angeles" projects/fb_social/ → ни одной живой (только комментарии-история)

## 2. ПАУЗА ПОЛОСЫ ТЕПЕРЬ ВАЛИТ ВСЕГО ОГНЯ — ДОКАЗАНО

**Почему не доставала.** Пауза живёт в `run_guarded.sh` и гейтит по МЕТКЕ крона (`*warmup*`).
EV-посты запускаются мимо `run_guarded` вообще — потому и проходили. (Уточнение к утренней
карте: я написал «пауза живёт в agent_capacity:473». Это правда для ПРОБУЖДЕНИЙ агентов, но
X-прогрев ловится не там, а в run_guarded. На вывод не влияет, но точность важна.)

**Живой прогон реальной фразой Босса** — `dirigent_commands.interpret("приостанови прогрев")`
→ `{'intent': 'pause_lane', 'lane': 'warmup'}` → пульт `warmup: paused`:

    1. ev_group_recurring_post.sh  → SKIP: «полоса на ПАУЗЕ Босса» — до запуска python
    2. fb_active.py session        → skip_gate — телефон не тронут
    3. fb_zdravurrrr_warmup.run()  → skip_gate — телефон не тронут
    4. run_guarded.sh xwarmup      → SKIP — X тоже встал

**Стейт не сожжён** (это был баг 05.07: заход вне окна двигал ротацию без поста) —
md5 обоих файлов до и после прогона идентичны:

    23169dce3faa3fb2cd3ce086712fd869  ev_recurring_last     (не изменился)
    31acd06cd99be96d8cd8aee52cbc8025  fb_active_state.json  (не изменился)

**«возобнови прогрев»** → `warmup: live` → гейт снова 🟢, X через run_guarded снова проходит,
fb_active реально пошёл листать ленту (см. лог выше). Пауза обратима, проверено в обе стороны.

**Аварийный стоп намеренно НЕ продублирован** в ogon_gate: он живёт на горле `tab_lock`
(fail-closed), через которое fb_active и так проходит. Второе место = второй шанс разъехаться.
Одно горло — один стоп.

## 3. DAYTIME-EXEC — ТЕКСТ БОЛЬШЕ НЕ МОЖЕТ ВРАТЬ

Не стал ни переписывать литерал на «06-22 ET», ни удалять.

* **Переписать** = завести ту же мину на следующую смену окна: через месяц окно поменяют в
  `warmup_window`, а строка в кронтабе опять останется старой и опять будет врать. Ровно так
  и родилась вся эта работа.
* **Удалить** = молча отнять у Босса утреннее напоминание, которое стоит там не просто так

    1) ВЕЛИК — приоритет: описание в листинге + кросс-пост в 2-3 локальные OC bike-группы...

Окно берётся из `warmup_window`, состояние — живой вердикт `ogon_gate`. Поставишь прогрев на
паузу — нудж честно напишет «⛔ Огонь молчит», а не позовёт бодро постить EV. Врать бодростью —

Заодно поправил два кронтаб-комментария, ещё описывавших Pacific-режим («ТОЛЬКО день Pacific»,
