# Операции начаты: механизм взведён, гвард починен, первый лид оказался БЕЗ СООБЩЕНИЯ

## 0. Модель для ответов покупателю — подтверждено ФАКТОМ
Путь: `generate_reply()` → `_sanitize_reply(_llm_text(prompt))` →
`subprocess.run([claude, "-p", prompt, "--model", "claude-opus-4-8"])`.
**Проверено живьём, не по конфигу:** CLI `/usr/bin/claude` отвечает, вызов идёт с
`--model claude-opus-4-8`.

**Честно про исключения — не каждый ответ пишет модель:**
- фолбэка на слабую модель НЕТ: Haiku-API откачен (он делил ключ Титана). Нет CLI → пустой
  ответ → лид **эскалируется**, шаблон не отправляется;
- но ДВА гварда ПОДМЕНЯЮТ текст модели фиксированным шаблоном:
  вещь отдана, а модель этого не сказала → `SAFE_GONE_REPLY`;
  модель пытается отшить покупателя на АКТИВНОМ листинге → `SAFE_BIKE_REPLY`/`SAFE_GENERIC_REPLY`.
- есть до 2 попыток генерации, если модель выдала мета-текст («Let me re-read…»).

## 1. `lead_guarantee` ВЗВЕДЁН
`ARMED=True`. `MAX_PER_RUN=1` — один лид за прогон, один экран телефона.

## 🔴 2. ПЕРЕД ЗАПУСКОМ ПОЙМАН СЛОМАННЫЙ ГВАРД (Ирине написали бы вопреки приказу)
`is_parked()` сравнивал `"irena irena"` (ПРОБЕЛ) с `"lead-fb-sales-irena-irena"` (ДЕФИСЫ) —
подстрокой не входит, парковка промахивалась **молча**, Ирина снова числилась сиротой.
Взведённый механизм поднял бы на неё Сеню — против твоего прямого приказа.

**Первый фикс был хуже болезни** и тоже пойман проверкой: нормализация через `[^a-z0-9]`
срезала кириллицу, `«мультиварка»` стала ПУСТОЙ строкой, а пустая входит в любую → гвард
объявил запаркованными ВСЕХ, сироты исчезли. Сторож, блокирующий всё, охраняет не лучше
пропускающего всё.

**Итог (коммит `dde8427`):** нормализация `isalnum()` + пустой ключ не матчит.
Проверено: Michelle `parked=True`, Ирина `parked=True`, сироты — sms-лид, Audrey, Larry.
