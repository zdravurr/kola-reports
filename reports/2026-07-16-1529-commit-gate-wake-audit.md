## ⚠️ СНАЧАЛА — ЧЕСТНЫЙ ПРЕДЕЛ ЭТОГО ШАГА

Это **не защита**. Агент с рутом обходит хук одним движением:
```
git commit --no-verify          # флаг
rm .git/hooks/pre-commit        # снёс
git -c core.hooksPath=/dev/null # мимо
```
Гейт ловит **НОРМАЛЬНЫЙ автономный коммит** — ровно ту модель угрозы, которую ты задал:
**полезный агент чинит то, что трогать нельзя**. Против такого он работает. Против
злоумышленника — ноль. **Жёсткая гарантия только после де-рутинга (следующий шаг).**
Не называть это безопасностью и не расслабляться.

### Торговля — защита ПО СОДЕРЖИМОМУ, потому что иначе никак

**Важный факт, который меняет постановку: Titan и Solana НЕ ЛЕЖАТ В ЭТОМ РЕПО.**
```
git ls-files | grep -cE 'titan|mercury|trades\.db'  →  0
```
Значит хук **физически не может** гейтить коммиты в них — таких коммитов здесь не бывает.
Твой пункт «anything touching /root/titan-bot» выполним только так: ловить момент, когда
**код из воркспейса начинает ТЯНУТЬСЯ в торговлю**:
```
/root/titan-bot | mercury-sol | /root/trades.db | titan.service | bingx_trade | mercury_sol
```
Именно так и выглядел бы «полезный» агент, решивший подправить бота. **Доказано живьём:**
файл в `scripts/` (НЕ защищённый путь) заблокирован **только за то, что дотянулся до Титана**.

### Поведение
- **БЛОК** → `exit 1`, коммита нет.
- **ТРЕВОГА** Боссу через `notify_boss` — файлы + сводка диффа.
- **УДЕРЖАНИЕ, а не потеря**: правка остаётся **в индексе** + патч в
  `.kola_state/commit_gate_held/<ts>.patch`. Молча не роняем НИЧЕГО.
- **Свободно**: `LEARNINGS.md`, логи, доки, отчёты — без токена, без шума.
- **Обход для своих**: `KOLA_COMMIT_GATE_TOKEN` = `secrets.json:commitGateToken`
  (сгенерил, лежит **вне репо**; значение нигде не печатал).

## 2. АУДИТ ПРОБУЖДЕНИЙ — И ПОПРАВКА К МОЕМУ ЖЕ ВЧЕРАШНЕМУ УТВЕРЖДЕНИЮ

**Я сказал: «аудит пробуждений дырявый». Это было НЕТОЧНО.** В логе **559** строк
`wake_main`. Правда тоньше и хуже:

**`agent_capacity.wake()` — обязательное горло — САМ в аудит не писал.** В аудит попадали
только те, кто логировал **себя сам** (`commit_watchdog`). А все, кто честно шёл через горло

**Фикс:** одна строка **в горле**, после успешного спавна (не до — аудит не должен врать о
том, чего не случилось). Теперь покрыты **все** вызывающие, по конструкции.

**Нашёл третьего спавнера:** `scripts/main_liveness.py` поднимает `openclaw agent`

## 🔴 НАХОДКА ПО ДОРОГЕ (не чинил — это твой следующий шаг)

**`commit_watchdog` будит main В ОБХОД `agent_capacity.wake()`** — прямым
`subprocess.run(["openclaw","agent",…])`. Он проверяет только память вручную. Значит мимо
него проходят: **потолок агентов (6), MAX_BURST, пауза полосы Дирижёра — и сам `GLOBAL_STOP`**.
То есть **твой рубильник его не остановит** (когда ты его вернёшь). Ровно то, что запрещает
докстринг `agent_capacity`: «прямой вызов = дыра, через которую 13.07 утекла вся память».
`main_liveness` — так же. **Не трогал**: рубильник — следующий шаг, скажешь — закрою.
