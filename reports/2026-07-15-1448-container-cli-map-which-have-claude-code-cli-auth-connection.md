## ОТВЕТЫ НА 4 ВОПРОСА

**1. Что каждый и зачем.**
- `dirigent` — живой оркестратор продаж/закупок (loop.sh принимает лид→полоса→диспетч Сене/Боре→ревью). CLI НЕТ.
- `ts-dirigent` — tailscale-сайдкар, даёт dirigent сеть (dirigent сидит в его netns).
- `claude-agent-1/2/3` — 3 контейнера с CLI из OpenClaw-мультиагента (собраны 07-09). СЕЙЧАС это простаивающие «husk»: CMD=`sleep infinity`, внутри только tini+sleep; НИКТО в них работу не запускает (ноль ссылок в кронах/скриптах).

**2. Есть ли у 3 claude-agent CLI + аутентификация + Opus 4.8? ДА, доказано живьём.**
- Запустил в claude-agent-1: `claude -p "..." --model claude-opus-4-8` → ответ **«ALIVE-OPUS»**. Работает СЕЙЧАС.
- У них выставлен `ANTHROPIC_MODEL=claude-opus-4-8`, node v24, creds .credentials.json.
- Токен истекает через ~4.8ч, но хостовый крон `oauth_refresh.py` (*/15) его обновляет.
- ⚠️ ВАЖНО: все 3 делят ОДИН токен — creds md5 идентичны, аккаунт `<аккаунт Босса, один Max>`.

**3. Есть ли CLI у dirigent? НЕТ — это и есть тот «контейнер без CLI».**
- Отдельный, более НОВЫЙ образ (07-14 против 07-09 у агентов), другой пользователь (`dirigent` vs `agent`), нет node, нет creds. Именно его я имел в виду, когда сказал «в контейнере нет claude CLI».

**4. dirigent-оркестратор и 3 claude-agent — одно или разное? Использует ли Дирижёр их? РАЗНОЕ, и НЕТ.**

- Значит Дирижёр их НЕ использует; они крутятся ОТДЕЛЬНО и простаивают. NetworkMode dirigent = `container:<ts-dirigent>`; агенты = `claude-net`. Все Restart=unless-stopped (переживут ребут).
