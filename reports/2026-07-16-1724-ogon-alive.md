# 2. ПЕТЛЯ ПРОГРЕВ→ЛИД→ПРОГРЕВ ЗАМКНУТА ✅

**Нашёл настоящую дыру — код сам в ней признавался:**
```python
if _warmup_should_yield_to_money():
    break        # ← и всё. Заход умирал.
# коммент: «прогрев не резюмится посреди — стартует заново следующим кроном»
```
То есть **один 3-минутный ответ Сени убивал весь заход Огня — до следующего крона, до 6 часов.**
Это не «уступил», это «ушёл до завтра».

**Стало:** уступка отдаёт **ЖЕЛЕЗО** (снимаем `tab_lock`), ждём ухода клиента, **забираем
телефон обратно и продолжаем тот же заход**.

**Доказано живьём — ключевая строка t+5s:**
```
t+0s   ПРОГРЕВ держит телефон        | yield?=False
t+0s   💰 ЛИД ПРИШЁЛ                  | yield?=True
t+5s   🟢 СЕНЯ СМОГ ВЗЯТЬ ТЕЛЕФОН → уступка НАСТОЯЩАЯ, не на словах
t+12s  ✅ Сеня отработал лид, метку снял
[yield] телефон был свободен для money-path 12с
t+12s  ПРОГРЕВ ВЕРНУЛСЯ = True   → ✅ ПЕТЛЯ ЗАМКНУТА
```
Сеня **физически смог взять flock** во время уступки — значит телефон реально освобождался.
Fail-safe: не смог отпустить/забрать или вышло окно сессии → заход честно заканчивается
(как раньше). Телефон при ожидании **не держим никогда**.
