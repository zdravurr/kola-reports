**Цена:** гейтвей — это путь пробуждения агентов. 27 минут детекторы видели инбоксы, но

**Урок:** `json.dump` в secrets — это не «запись значения», это **смена прав файла**.
Сохранять режим явно. Бэкап секретов снял до правки.

**Отдельно честно:** де-рутинг тут **ни при чём** — авария началась в 15:22, де-рутинг

# ФАЗА 4 — ДЫРА ЗАКРЫТА (та самая, что я доказал утром)

| Проверка | Было (утром) | Стало |
|---|---|---|
| `sudo -u botuser sudo -n id` | **uid=0(root)** | `sudo: a password is required` |
| `sudo -l -U botuser` | `(ALL) NOPASSWD: ALL` | **`User botuser is not allowed to run sudo`** |
| читать `/root/titan-bot/main.py` | **дотянулся** | `sudo: password required` / `Permission denied` |
| `trades.db` | достижим | **не читается** |
| `systemctl restart titan.service` | смог бы | `Interactive authentication required` |
| `visudo` / `su -` (рут-шелл) | смог бы | отказ |

**Точность формулировки:** у botuser **есть** хеш пароля (`$y$…`), поэтому «требуется
пароль» — не барьер сам по себе. Барьер — **авторизация**: правил нет вообще
(`not allowed to run sudo`), поэтому и с верным паролем sudo ничего не даст.

**Админ цел:** `(ALL : ALL) ALL` + `(ALL) NOPASSWD: ALL`, Титан читается.
